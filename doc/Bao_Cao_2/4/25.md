# BÁO CÁO KỸ THUẬT: KHẮC PHỤC SỰ CỐ MONGODB WINDOWS SERVICE

**Dự án:** Hệ thống SmartHome (Backend: FastAPI, App: Flutter)  
**Môi trường:** Windows 11, MongoDB Community Server cài trên ổ đĩa mở rộng (`E:\`)

---

## 1. Hiện trạng và Triệu chứng lỗi (Bug Logic)

Hệ thống Cơ sở dữ liệu (MongoDB) từ chối khởi động thông qua Windows Services, báo cáo hàng loạt mã lỗi dây chuyền:

* **Error 2186 / 1053:** Service bị treo (*Not responding*). Hệ thống gọi file thực thi `mongod.exe` nhưng không có kịch bản cấu hình (`.cfg`) dẫn đường, khiến tiến trình tự động đóng (crash ngầm) ngay lập tức.
* **Error 1058 / 1072:** Trạng thái "Zombie Service". Tiến trình cũ bị đánh dấu xóa (*marked for deletion*) nhưng kẹt trong bộ nhớ, khóa mọi quyền can thiệp, khiến lệnh `--install` mặc định của MongoDB bị vô hiệu hóa ngầm.
* **Hậu quả:** Port `27017` không mở, Backend FastAPI từ chối kết nối, hệ thống SmartHome tê liệt. Bắt buộc phải treo cửa sổ CMD/PowerShell thủ công để duy trì Database.

## 2. Phân tích Nguyên nhân (Root Cause)

1. **Lỗi kẹt file Lock:** Do tắt máy đột ngột, file `mongod.lock` không được dọn dẹp, cộng thêm cấu trúc dữ liệu bị tổn thương nhẹ (đã khắc phục bằng lệnh `--repair`).
2. **Thiếu định hướng cấu hình:** Việc cài đặt custom sang thư mục `E:\setupD` không đi kèm file cấu hình chuẩn YAML, khiến Service chạy ngầm mất phương hướng.
3. **Xung đột cơ chế Windows:** Lệnh `--install` của MongoDB xung đột với cơ chế quản lý Service của Windows khi xử lý các đường dẫn chứa dấu cách hoặc phân quyền ổ đĩa mở rộng.

## 3. Luồng Thực thi Giải pháp (Khắc phục triệt để)

Thay vì phụ thuộc vào trình cài đặt tự động của MongoDB, giải pháp can thiệp bạo lực ở tầng lõi hệ thống (OS level) đã được áp dụng:

### Bước 1: Thiết lập cấu hình chuẩn (Config Initialization)
* Tạo thư mục lưu log hệ thống: `E:\setupD\log\`.
* Tự viết file cấu hình `mongod.cfg` chuẩn cú pháp YAML, lưu tại root thư mục cài đặt `E:\setupD\mongod.cfg` để thiết lập tọa độ data, log và port mạng cục bộ (`127.0.0.1:27017`).

### Bước 2: Ép hệ thống định nghĩa Service mới (Bypass Windows Registry)
Sử dụng công cụ `sc.exe` (Service Control) quyền lực nhất của Windows qua môi trường `cmd.exe` để tạo một Service hoàn toàn mới, độc lập với process cũ:

```powershell
cmd.exe /c 'sc create MongoDBSmartHome binPath= "\"E:\setupD\bin\mongod.exe\" --config \"E:\setupD\mongod.cfg\" --service" DisplayName= "MongoDB SmartHome" start= auto'
